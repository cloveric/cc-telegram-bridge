One Bot, One Instance

Every instance keeps its own token, access model, lock, inbox, logs, update watermark, and Codex threads.

Private-Chat Control

Pairing and allowlist rules gate execution before Codex work or attachment downloads are allowed to happen.

Resumable Coding Threads

The first real user message creates a Codex thread, and later messages resume that exact thread instead of starting cold.

Operational Model

Built for multiple bots without turning one process into a multiplexed mess.

This project chooses isolated instances over shared multi-tenant runtime state. If you need three bots, you run three instances. That keeps locks, logs, pairing data, sessions, and failure domains explicit.

Human Factors

Looks like chat. Behaves like tooling.

The Telegram side stays simple: pairing codes, placeholder messages, edited final responses, and uploaded files. The Codex side keeps service control, state, and reviewability where operators actually need it.

Lifecycle

02

03

Pair

04

Resume

Keep chat-to-thread bindings so later instructions continue the same Codex conversation.
